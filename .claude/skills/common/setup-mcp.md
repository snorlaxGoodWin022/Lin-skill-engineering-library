# MCP 自动化工作流配置

你是前端 MCP 工作流配置专家。请按以下步骤分析当前项目，为其配置最合适的 MCP Server，并生成自动化工作流规则。

用户附加要求: $ARGUMENTS

---

## 第一步：项目分析

请依次检查以下文件来识别项目技术栈：

1. **读取 `package.json`** — 识别框架、UI 库、状态管理、构建工具、测试框架
2. **检查配置文件** — `vite.config.*`、`next.config.*`、`nuxt.config.*`、`webpack.config.*`、`tsconfig.json` 等
3. **扫描 `src/` 目录结构** — 了解项目的分层架构
4. **检查是否已有 `.mcp.json` 或 `CLAUDE.md`** — 避免覆盖已有配置

分析完成后，向用户输出一份简明的技术栈报告（框架、UI 库、状态管理、构建工具、其他关键依赖）。

---

## 第二步：MCP Server 推荐

根据第一步的分析结果，按照以下决策树推荐 MCP Server：

### Tier 1 — 所有前端项目必装

| MCP Server     | 安装命令                     | 能力                                       |
| -------------- | ---------------------------- | ------------------------------------------ |
| **Playwright** | `npx @playwright/mcp@latest` | 浏览器自动化、截图验证、E2E 测试、页面交互 |

**推荐理由：** 前端开发的核心闭环是「写代码 → 看效果」。Playwright 让 Claude 能自动打开浏览器、截图、点击、验证页面，实现真正的自动化验证。

| MCP Server   | 安装命令                           | 能力                                |
| ------------ | ---------------------------------- | ----------------------------------- |
| **Context7** | `npx @upstash/context7-mcp@latest` | 实时查询任意库/框架的最新文档和 API |

**推荐理由：** AI 的训练数据有过期问题。Context7 确保生成的代码基于最新版本的 API，避免使用已废弃的方法。

### Tier 2 — 按场景推荐

根据以下条件判断是否推荐：

| 条件                     | 推荐 MCP         | 安装命令                                   |
| ------------------------ | ---------------- | ------------------------------------------ |
| 项目有 UI 设计稿对接需求 | **Figma MCP**    | 需要通过 `/mcp` 命令认证，不写入 .mcp.json |
| 项目使用 GitHub 托管     | **GitHub MCP**   | `npx @anthropic-ai/github-mcp@latest`      |
| 项目需要网页内容抓取     | **Fetch MCP**    | `npx @anthropic-ai/fetch-mcp@latest`       |
| 项目对接 Supabase 后端   | **Supabase MCP** | `npx @supabase/mcp-server-supabase@latest` |
| 项目使用 Netlify 部署    | **Netlify MCP**  | `npx @netlify/mcp-server@latest`           |

### 不推荐安装的情况

- 项目纯后端（Node.js / Python / Go）→ 只保留 Context7，去掉 Playwright
- 项目是静态文档站点 → 只保留 Context7
- 项目已有完善的测试体系 → 仍推荐 Playwright（用于视觉验证，不是替代现有测试）

## 第四步：创建 .mcp.json

在项目根目录创建 `.mcp.json`，只包含确认后的 MCP Server。

格式如下：

```json
{
  "mcpServers": {
    "playwright": {
      "command": "npx",
      "args": ["-y", "@playwright/mcp@latest"]
    },
    "context7": {
      "command": "npx",
      "args": ["-y", "@upstash/context7-mcp@latest"]
    }
  }
}
```

注意事项：

- 如果项目已有 `.mcp.json`，**合并而非覆盖**已有配置
- Figma MCP 需要用户手动认证，不写入 .mcp.json，改为提示用户后续通过 `/mcp` 命令添加
- 如果用户的项目不需要某个 MCP，不要写入

## 第五步：生成/更新 CLAUDE.md

在项目根目录创建或**更新** `CLAUDE.md`。

如果已有 CLAUDE.md，**在文件末尾追加**以下内容，不要覆盖已有内容。
如果已有 CLAUDE.md 中已有 `## MCP 自动化工作流` 章节，**替换该章节**。

### CLAUDE.md 模板

根据实际配置的 MCP，生成对应的规则。以下为完整模板（只包含实际安装的 MCP）：

```markdown
## MCP 自动化工作流

本项目已配置以下 MCP Server。Claude 应根据任务自动选择使用，无需用户手动指定。

### Playwright MCP — 浏览器自动化 & 视觉验证

**自动触发场景：**

- 用户说"看看效果"、"验证一下"、"打开页面"、"截图"时
- 完成组件/页面开发后，用户要求检查效果时
- 用户要求进行流程测试（如"测试登录流程"）时
- 用户要求对比设计与实现的差异时

**使用流程：**

1. 确认 dev server 已启动（如未启动，先运行启动命令）
2. 通过 Playwright 打开本地开发地址（默认 http://localhost:5173，根据实际端口调整）
3. 导航到目标页面，执行操作或截图
4. 将截图结果反馈给用户，指出发现的问题

### Context7 MCP — 最新文档查询

**自动触发场景：**

- 用户问到项目中使用的库/API 的用法时
- 生成代码时需要确认某个 API 的最新签名时
- 用户说"查一下"、"怎么用"、"最新"时
- 使用不确定是否已变更的 API 时

**使用流程：**

1. 查询相关库的最新文档
2. 基于返回的文档内容生成准确的代码
3. 如果发现项目中的代码使用了已废弃的 API，主动提醒用户

### Figma MCP — 设计稿转代码

**自动触发场景：**

- 用户分享 Figma 链接要求实现时
- 用户说"按设计稿实现"、"还原设计"时
- 需要确认 UI 规范（颜色、间距、字体）时

**前置条件：** 需要先通过 `/mcp` 命令认证 Figma 账号。
```

## 重要原则

1. **不要过度配置** — 只推荐真正能提效的 MCP，不堆砌数量
2. **合并而非覆盖** — 已有的 .mcp.json 和 CLAUDE.md 要合并，不丢失内容
3. **基于事实推荐** — 所有推荐必须基于第一步的实际分析，不猜测
4. **尊重用户选择** — 第三步必须等待用户确认后再创建文件
5. **给出具体示例** — 示例指令要基于项目的实际页面和组件，不用泛泛的例子
