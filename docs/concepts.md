# Skill.md 概念介绍

本文档介绍 Skill.md 的核心概念、工作原理和设计理念。

## 什么是 Skill.md？

Skill.md 是一种给 AI 提供"工作手册"的标准化文档格式。就像入职新公司时拿到的《开发规范文档》，Skill.md 告诉 AI：

- **项目如何组织代码** - 目录结构、文件命名
- **有哪些约定俗成的规矩** - 编码规范、命名习惯
- **常见业务怎么处理** - 表单、CRUD、错误处理
- **代码写成什么样才算合格** - 质量标准、测试要求

**核心价值：一次配置，持续复用。**

## Skill.md vs Prompt 的区别

| 对比维度 | Prompt | Skill.md |
|----------|--------|----------|
| 使用时机 | 每次对话都要写 | 写一次，长期使用 |
| 内容性质 | 具体任务描述 | 通用规范和知识 |
| 适用场景 | "帮我生成一个登录页" | "我们项目所有页面的开发规范" |
| 更新频率 | 随需求变化 | 技术栈变更时才改 |
| 典型长度 | 100-500字 | 1000-5000字 |
| 维护成本 | 高（每次都要写） | 低（一次编写） |
| 复用性 | 低 | 高（跨项目可用） |

## Skill.md 的核心组成部分

### 1. 使用场景
描述该 Skill 适用于什么情况，帮助 AI 判断何时使用。

```markdown
## 使用场景
当需要创建任何数据录入表单时使用，包括：
- 用户信息表单
- 订单创建表单
- 设置配置表单
```

### 2. 技术栈约定
明确项目使用的技术和版本。

```markdown
## 技术栈约定
- React 18.2
- TypeScript 5.0
- Ant Design 5.x
- react-hook-form 7.x
- Zod（表单校验）
```

### 3. 代码规范
命名规范、文件结构、编码风格等。

```markdown
## 命名规范
- 组件文件：PascalCase.tsx
- 变量命名：camelCase
- 常量命名：UPPER_SNAKE_CASE
```

### 4. 业务模板
常见业务场景的代码模板。

```markdown
## 组件结构模板
每个表单组件必须按以下结构组织：
FormName/
├── index.tsx           # 组件主文件
├── schema.ts           # Zod 校验规则
├── types.ts            # TypeScript 类型定义
└── hooks/              # 自定义 Hooks
```

### 5. 输出要求
对 AI 生成代码的质量要求。

```markdown
## 输出要求
1. 类型定义完整，无 any 类型
2. 包含所有字段的校验规则
3. 提交逻辑完整（loading、错误处理）
4. 至少 3 个测试用例
```

## 工作原理

当 AI 接收到用户请求时：

1. **识别需求** - 用户提到"按照 xxx 规范"
2. **查找文件** - 在 `.claude/skills/` 目录寻找对应的 `.skill.md` 文件
3. **读取理解** - 解析 Skill 文件的结构和要求
4. **生成代码** - 按照 Skill 中的约定生成代码
5. **自我检查** - 确保生成的代码符合 Skill 要求

### 示例流程

```
用户：请按照 form-generator.skill.md 规范，生成用户注册表单

AI 执行步骤：
1. 读取 form-generator.skill.md
2. 了解技术栈：React + TypeScript + Ant Design + react-hook-form + Zod
3. 按照文件结构创建：UserRegisterForm/index.tsx, schema.ts, types.ts
4. 根据规范添加字段：用户名、邮箱、密码
5. 添加校验规则：必填、格式、长度限制
6. 添加提交逻辑：loading 状态、错误处理
7. 输出完整代码
```

## 为什么需要 Skill.md？

### 解决的问题

| 问题 | 传统方式 | 使用 Skill.md |
|-----|---------|--------------|
| 重复沟通 | 每次都要告诉 AI 技术栈 | AI 自动遵循 |
| 代码不一致 | 不同时间生成的代码风格各异 | 风格统一 |
| 质量参差 | AI 生成代码质量不稳定 | 符合团队标准 |
| 新人上手慢 | 需要大量文档学习 | Skill.md 即规范 |

### 带来的好处

- **效率提升** - AI 生成代码更准确，减少返工
- **质量保证** - 代码符合团队标准
- **知识沉淀** - 团队经验固化到文档
- **协作顺畅** - 所有人使用同一套标准

## 适用场景

| 场景 | 适用性 | 说明 |
|-----|-------|------|
| 个人项目 | ✅ 适合 | 保持代码风格一致，复用成功经验 |
| 团队开发 | ✅ 适合 | 统一编码规范，新成员快速上手 |
| 开源项目 | ✅ 适合 | 贡献者指南，规范代码风格 |
| 一次性任务 | ❌ 不适合 | 直接用 Prompt 更高效 |
| 探索性项目 | ❌ 不适合 | 规范可能频繁变化 |

## 局限性

1. **学习成本** - 需要学习编写格式，整理团队规范
2. **维护成本** - 技术栈更新时需要同步更新
3. **适用边界** - 不适合快速原型验证阶段

## 最佳实践

1. **渐进式采用** - 从一个 Skill 开始，验证效果后逐步扩展
2. **版本控制** - Skill.md 纳入 Git 管理，记录更新日志
3. **团队协作** - 定期评审 Skill，收集使用反馈
4. **文档配套** - 提供使用示例和操作指南
